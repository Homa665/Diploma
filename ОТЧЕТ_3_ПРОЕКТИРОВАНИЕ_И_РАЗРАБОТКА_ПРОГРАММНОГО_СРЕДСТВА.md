# 3 ПРОЕКТИРОВАНИЕ И РАЗРАБОТКА ПРОГРАММНОГО СРЕДСТВА

## 3.1 Проектирование и разработка пользовательского интерфейса

При анализе реализованного проекта установлено, что интерфейс разработан как серверно-рендерируемое веб-приложение на HTML-шаблонах с единым CSS-стилем и JavaScript-слоем для интерактивности. Клиентская часть находится в каталоге `web` и состоит из:

- `web/templates` — набор экранов (главная, лента, профиль, пост, чат, уведомления, админ-панель и др.).
- `web/static/css/style.css` — централизованная дизайн-система.
- `web/static/js/app.js` — поведение UI: формы, уведомления, чат-поллинг, бесконечная прокрутка, модальные окна, рейтинг.

### Реализованные экраны и пользовательские сценарии

В проекте реализованы следующие интерфейсные страницы:

- Гостевые/входные: `home.html`, `login.html`, `register.html`.
- Основные пользовательские: `feed.html`, `post.html`, `create_post.html`, `edit_post.html`.
- Коммуникации: `chat.html`, `chat_with.html`, `notifications.html`.
- Профиль и развитие аккаунта: `profile.html`, `settings.html`, `premium_stats.html`, `expert_apply.html`, `pricing.html`.
- Командная работа и проекты: `teams.html`, `project.html`, `consultations.html`, `consultation.html`.
- Администрирование: `admin.html`.

Навигация унифицирована и зависит от роли пользователя (`user`, `premium`, `expert`, `admin`). Для авторизованных доступны лента, создание постов, чат, уведомления, профиль, статистика и административные функции (для `admin`).

### Интерактивность и UX-логика

В `app.js` реализованы:

- API helper с обработкой ошибок и универсальной отправкой форм.
- Toast-уведомления и push-like уведомления о новых событиях.
- Polling уведомлений (`/api/notifications`) каждые 15 секунд.
- Polling чата (`/api/messages`) каждые 3 секунды.
- Infinite scroll в ленте (`/api/feed?page=...`).
- Рейтинг через интерактивные звезды.
- Вкладки, модальные окна, универсальные action-кнопки.

Таким образом, интерфейс обеспечивает полный цикл работы пользователя: регистрация, публикации, коммуникации, модерация, развитие статуса и работа в команде.

## 3.2 Выбор и обоснование технологий

Фактический стек проекта (по коду):

| Слой | Технологии |
|---|---|
| Backend | Go, `net/http`, middleware, html/template |
| База данных | PostgreSQL |
| Доступ к БД | `database/sql` + драйвер `pgx/v5/stdlib` |
| Аутентификация | JWT (секрет из ENV) |
| Frontend | SSR шаблоны HTML + CSS + Vanilla JavaScript |
| Деплой/запуск | `go run ./cmd/server/`, бинарная сборка |
| Тестирование | Go tests (`./tests`) |

### Обоснование по фактической реализации

1. **Go + net/http**: простая и производительная серверная архитектура без перегрузки зависимостями.
2. **PostgreSQL**: надежное реляционное хранилище для связанного социального графа (посты, комментарии, подписки, чаты, жалобы).
3. **SQL-first миграции в коде**: все таблицы и индексы создаются централизованно в `internal/database/database.go`.
4. **SSR-подход**: снижает сложность фронтенда, упрощает SEO и рендер первых экранов.
5. **Vanilla JS**: достаточен для интерактивности (поллинг, модалки, формы, бесконечная лента).

## 3.3 Разработка архитектуры программного средства

Проект реализован в многоуровневой серверной архитектуре с разделением ответственности.

### Структура серверной части

| Каталог | Назначение |
|---|---|
| `cmd/server` | Точка входа, сборка маршрутов, запуск HTTP-сервера |
| `internal/config` | Конфигурация из переменных окружения |
| `internal/database` | Подключение к БД, миграции, индексы, эволюция схемы |
| `internal/seed` | Начальное заполнение БД данными |
| `internal/models` | DTO/модели домена |
| `internal/middleware` | Auth/API/Auth Optional/Admin middleware |
| `internal/handlers` | Бизнес-логика HTTP-обработчиков |
| `web/templates` | SSR-шаблоны страниц |
| `web/static` | CSS/JS/статические ресурсы |

### Ключевые функциональные модули (handlers)

| Файл | Функциональность |
|---|---|
| `auth.go` | Регистрация, вход, выход |
| `posts.go` | Лента, посты, файлы, рейтинг, опросы |
| `comments.go` | Комментарии и лайки комментариев |
| `chat.go` | Личные сообщения и обмен контентом |
| `profile.go` | Профили, подписки, проекты, экспертность, премиум, настройки |
| `admin.go` | Модерация, жалобы, баны, роли, удаление |
| `handler.go` | Общие функции, работа с шаблонами |

### Архитектурные особенности

- Централизованный роутинг в `cmd/server/main.go`.
- Разграничение доступа через middleware по роли и авторизации.
- Работа с БД через SQL-запросы и индексацию под основные сценарии.
- Унифицированная визуальная система и повторно используемые UI-паттерны.
